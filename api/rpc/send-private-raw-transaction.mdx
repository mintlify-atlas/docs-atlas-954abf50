## Overview

Sends a signed raw transaction privately to multiple block builders, protecting against frontrunning and sandwich attacks. The transaction is kept private until it is included in a block.

<Note>
This method broadcasts your transaction to all available builders simultaneously. If at least one builder successfully accepts the transaction, the method returns the transaction hash.
</Note>

## Request parameters

<ParamField path="tx" type="Bytes" required>
  The signed raw transaction data as a hex-encoded string with `0x` prefix.
  
  This should be a fully signed transaction that has been encoded according to Ethereum's transaction encoding rules (RLP-encoded).
</ParamField>

## Response

<ResponseField name="result" type="B256">
  The transaction hash as a 32-byte hex string with `0x` prefix.
  
  This hash uniquely identifies the transaction and can be used to track its status.
</ResponseField>

<Warning>
The method succeeds if **at least one** builder accepts the transaction. This means some builders may reject it while others accept it, and the method will still return successfully.
</Warning>

## Implementation details

When you call this method, the following sequence occurs:

1. **Builder initialization** - All available builders are instantiated from the `BuilderKind` enum (`main.rs:71-83`)
2. **Transaction recovery** - The raw transaction is decoded and the hash is computed (`main.rs:103-104`)
3. **Parallel broadcast** - The transaction is sent to all builders simultaneously using `join_all` (`main.rs:86`)
4. **Success validation** - If at least one builder succeeds, the transaction hash is returned (`main.rs:88-92`)

<Note>
The transaction hash is computed locally before sending to builders, so the returned hash is guaranteed to be correct regardless of which builders accept the transaction.
</Note>

## Use cases

- Protecting high-value transactions from frontrunning
- Preventing sandwich attacks on DEX trades
- Executing time-sensitive transactions with privacy
- Submitting MEV-sensitive operations without public mempool exposure
